07. JSON Plano vs. RESTful HATEOAS

API Tradicional (JSON)

    {
      "id": 101,
      "user": "Gabriel",
      "status": "active"
    }
El cliente debe "adivinar" la URL para editar o borrar.

API con HATEOAS (Fielding)

    {
      "id": 101,
      "user": "Gabriel",
      "links": [
        {"rel": "self", "href": "/users/101"},
        {"rel": "delete", "href": "/users/101", "method": "DELETE"},
        {"rel": "update", "href": "/users/101", "method": "PUT"}
      ]
    }
El servidor guía al cliente sobre qué puede hacer.
