03. NodeJS: I/O No Bloqueante vs. Apache

A diferencia de servidores como Apache HTTP Server que asignan un "Thread" (hilo) por cada conexión (limitado por la RAM), Node utiliza libuv.

  • Single Threaded: No hay cambios de contexto de CPU entre hilos.
  • Asincronía: Las operaciones de red o disco se delegan al SO, liberando el bucle principal.
Referencia Académica:

Tilkov, S., & Vinoski, S. (2010). Node.js: Using JavaScript to Build High-Performance Network Programs. IEEE Internet Computing.
